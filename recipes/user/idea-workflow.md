# Wingman Idea Workflow

This recipe implements a comprehensive workflow for recording new ideas, classifying them as new products or features, and automatically setting up the appropriate development environment.

## Workflow Steps

1. **Idea Recording & Classification**
   - Capture idea details in Obsidian
   - Classify as: New Product, Existing Product Feature, or Research/Exploration
   - Determine project scope and requirements

2. **Environment Setup**
   - For new products: Create new product suite structure
   - For existing product features: Create worktree branch
   - Setup Obsidian documentation structure

3. **Initial Planning**
   - Create project roadmap
   - Setup development tasks
   - Link documentation to code

## Usage

Run this recipe when you have a new idea to explore. The agent will guide you through the classification and setup process.

## Prerequisites

- Obsidian vault configured
- Git repository with worktree support
- Product suite scripts available

## Implementation

The workflow uses a combination of:
- Obsidian MCP for documentation
- Git worktree management
- Automated project setup scripts
- Structured templates
